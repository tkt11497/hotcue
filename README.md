# GCN Voice

Offline LAN voice communication app using WebRTC, Vue 3, and Socket.io.

## Architecture

```
┌──────────────┐    WebRTC (P2P audio)    ┌──────────────┐
│   Browser A  │◄────────────────────────►│   Browser B  │
│   (Vue 3)    │                          │   (Vue 3)    │
└──────┬───────┘                          └──────┬───────┘
       │  Socket.io (signaling)                  │
       └──────────────┬─────────────────────────┘
                      │
              ┌───────▼────────┐
              │  Node.js HTTPS │
              │  Signaling Srv │
              └────────────────┘
```

- **Signaling**: Node.js + Socket.io over HTTPS handles room management and WebRTC offer/answer/ICE exchange
- **Voice**: WebRTC peer-to-peer connections carry audio directly between browsers
- **Certificates**: mkcert generates locally-trusted HTTPS certs (required for microphone access)

## Prerequisites

- [Node.js](https://nodejs.org/) v18+
- [mkcert](https://github.com/FiloSottile/mkcert) — install with `choco install mkcert` or `scoop install mkcert`

## Quick Start

### 1. Generate HTTPS certificates

```bash
# Windows
setup-certs.bat

# macOS / Linux
chmod +x setup-certs.sh && ./setup-certs.sh
```

This installs a local CA and creates `server/certs/cert.pem` + `server/certs/key.pem`.

### 2. Install dependencies

```bash
cd server && npm install
cd ../client && npm install
```

### 3. Start the signaling server

```bash
cd server
npm start
```

Server runs at `https://localhost:3001`.

### 4. Start the frontend

```bash
cd client
npm run dev
```

Frontend runs at `https://localhost:5173`.

### 5. Connect

1. Open `https://localhost:5173` in your browser
2. Enter a display name and room name
3. Click **Join Room**
4. Open the same URL on another device on the same LAN (use the host machine's IP, e.g. `https://192.168.1.x:5173`)

## LAN Access

Other devices on the same network can connect using the host machine's local IP address:

- Frontend: `https://<HOST_IP>:5173`
- Update the **Server URL** in the lobby to: `https://<HOST_IP>:3001`

> **Note**: Other devices may see a certificate warning since the mkcert CA is only trusted on the host machine. They can proceed past the warning (the connection is still encrypted).
